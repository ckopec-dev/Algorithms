# Pollard's Rho Algorithm in Lua

Here's an implementation of Pollard's Rho algorithm for integer factorization in Lua:

```lua
-- Pollard's Rho Algorithm Implementation
function pollard_rho(n)
    if n % 2 == 0 then
        return 2
    end
    
    local x = 2
    local y = 2
    local c = 1
    local d = 1
    
    local function f(x)
        return (x * x + c) % n
    end
    
    while d == 1 do
        x = f(x)
        y = f(f(y))
        d = gcd(math.abs(x - y), n)
    end
    
    return d
end

-- Extended Euclidean Algorithm for GCD
function gcd(a, b)
    while b ~= 0 do
        a, b = b, a % b
    end
    return a
end

-- Function to find all prime factors
function factorize(n)
    local factors = {}
    
    if n <= 1 then
        return factors
    end
    
    while n > 1 do
        if n % 2 == 0 then
            table.insert(factors, 2)
            n = n // 2
        else
            local factor = pollard_rho(n)
            if factor == n then
                -- n is prime
                table.insert(factors, n)
                break
            else
                table.insert(factors, factor)
                n = n // factor
            end
        end
    end
    
    return factors
end

-- Example usage
print("Pollard's Rho Factorization Algorithm")
print("=====================================")

-- Test with a composite number
local test_number = 1387
print("Factoring: " .. test_number)

local factors = factorize(test_number)
print("Factors: " .. table.concat(factors, " × "))

-- Another example
local test_number2 = 1537
print("\nFactoring: " .. test_number2)
local factors2 = factorize(test_number2)
print("Factors: " .. table.concat(factors2, " × "))

-- Prime number test
local prime_test = 97
print("\nFactoring: " .. prime_test)
local factors3 = factorize(prime_test)
print("Factors: " .. table.concat(factors3, " × "))
```

## How it works:

1. **Algorithm Overview**: Pollard's Rho is a probabilistic factorization algorithm that works well for numbers with small prime factors.

2. **Key Components**:
   - Uses a pseudo-random sequence generated by a polynomial function
   - Applies the Floyd's cycle-finding algorithm to detect cycles
   - Uses GCD to find non-trivial factors

3. **Example Output**:
```

=====================================
Factoring: 1387
Factors: 19 × 73

Factoring: 1537
Factors: 23 × 67

Factoring: 97
Factors: 97
```

## Key Features:

- **Cycle Detection**: Uses Floyd's cycle-finding algorithm to detect when the sequence repeats
- **Polynomial Function**: Implements f(x) = x² + c (mod n) for generating the sequence
- **GCD Calculation**: Uses Euclidean algorithm to find the greatest common divisor
- **Edge Cases**: Handles even numbers and prime numbers properly

The algorithm has a time complexity of approximately O(n^(1/4)) and is particularly effective for finding small factors of large composite numbers.
